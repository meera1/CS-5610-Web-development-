Experiment 49: OAuth

Documentation

In this experiment we will learn some basics about OAuth.

What is OAuth? 

The idea behind OAuth is to provide a standard way for accessing to protected resources, 
             without the need of sharing passwords. You can do this by the use of tokens and one dosen't 
             need to share his/her credentials all the time

OAuth history 

OAuth was first designed as a protocol, to be interoperable and super easy to implement 
            for the developers. However, there are 30+ different implementations co-existing.

OAuth Working 

            Resource Owner: the user who wants to share a resource, e.g. owner of the facebook photos.
            Client: the application that wants to leverage a resource hosted by a third party, e.g. the photo printing website.
            Authorization Server: the entity that decides to grant access to the client (application), e.g. Facebook’s authorization server.
            Resource Server: the place where the third party resource is hosted, e.g. Facebook’s server where the photos to print are.

Conclusion 

We learnt what OAuth is and also learnt the working of it which is mandatory since OAuth is very complicated
